## User Story

**Como** Pablo (desarrollador) o integrador de sistemas  
**Quiero** procesar reuniones mediante API REST  
**Para** integrar MeetMind con otros sistemas o automatizar flujos

---

## Descripción ampliada

La API REST debe exponer endpoints para:

1. **Procesar archivo (multipart):** Subir archivo multimedia o texto y obtener resultados
2. **Procesar texto plano (JSON):** Enviar texto directamente sin archivo
3. **Listar reuniones:** Historial de reuniones procesadas
4. **Detalle de reunión:** Obtener una reunión por id
5. **Health check:** Verificar disponibilidad del servicio

Las respuestas deben usar códigos HTTP adecuados (400 Bad Request, 422 Validation Error, 500 Internal Server Error). La documentación OpenAPI (Swagger) debe estar disponible en `/docs`.

---

## Criterios de Aceptación

- [ ] POST /api/v1/process/file: procesar archivo (multipart)
- [ ] POST /api/v1/process/text: procesar texto plano (JSON body)
- [ ] GET /api/v1/meetings: listar historial
- [ ] GET /api/v1/meetings/{id}: detalle de reunión
- [ ] GET /health: health check
- [ ] Respuestas con códigos HTTP adecuados (400, 422, 500)
- [ ] Documentación OpenAPI disponible

```gherkin
    And el cuerpo incluye participants, topics, actions, minutes, summary

  Scenario: Procesar archivo vía multipart
    Given que tengo un archivo TXT válido
    When envío POST /api/v1/process/file con el archivo
    Then recibo 200 con resultados procesados

  Scenario: Error de validación - texto vacío
    When envío POST /api/v1/process/text con {"text": ""}
    Then recibo 400 o 422
    And el cuerpo indica el error de validación

  Scenario: Documentación OpenAPI
    When accedo a /docs
    Then veo la documentación Swagger/OpenAPI
```

## Especificación de Endpoints

| Método | Ruta | Body | Respuesta |
|--------|------|------|-----------|
| GET | /health | - | 200, `{"status": "ok"}` |
| POST | /api/v1/process/text | `{"text": "..."}` | 200, MeetingResult o 400/422 |
| POST | /api/v1/process/file | multipart/form-data | 200, MeetingResult o 400/422 |
| GET | /api/v1/meetings | - | 200, lista de MeetingRecord |
| GET | /api/v1/meetings/{id} | - | 200, MeetingRecord o 404 |
