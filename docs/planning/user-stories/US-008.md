## User Story

**Como** Marina o Pablo  
**Quiero** usar una interfaz web para subir archivos  
**Para** procesar reuniones sin usar la línea de comandos ni la API directamente

---

## Descripción ampliada

La interfaz Gradio debe ofrecer una experiencia completa para usuarios no técnicos:

- Componente de upload que acepte archivos multimedia (MP4, MOV, MP3, etc.) y texto (TXT, MD)
- Feedback visual durante la carga (progress, spinner)
- Feedback durante el procesamiento (mensaje "Procesando...")
- Visualización clara de resultados: participantes, temas, acciones, minuta, resumen
- Validación de tipos de archivo antes de enviar
- Mensajes de error claros y amigables cuando algo falla

La UI consume la API REST para procesar archivos y mostrar resultados.

---

## Criterios de Aceptación

- [ ] Interfaz de carga (upload) para archivos multimedia y texto
- [ ] Feedback visual durante la carga y procesamiento
- [ ] Visualización de resultados (participantes, temas, acciones, minuta, resumen)
- [ ] Validación de tipos de archivo soportados
- [ ] Mensajes de error claros si el archivo no es válido

## Especificación BDD

```gherkin
Feature: Subir archivos desde la interfaz web
  Como Marina o Pablo
  Quiero usar una interfaz web para subir archivos
  Para procesar reuniones sin usar la línea de comandos ni la API directamente

  Scenario: Cargar archivo válido
    Given que tengo un archivo TXT o MP3 válido
    When selecciono el archivo y pulso "Procesar"
    Then veo feedback visual durante la carga
    And veo indicador de procesamiento
    And recibo los resultados estructurados

  Scenario: Intentar cargar archivo no soportado
    Given que selecciono un archivo .exe o .zip
    When intento procesarlo
    Then recibo un mensaje de error claro
    And el sistema indica los formatos soportados

  Scenario: Procesamiento fallido
    Given que ocurre un error durante el procesamiento
    When la API retorna error
    Then muestro mensaje amigable al usuario
    And no expongo errores técnicos crudos
```
