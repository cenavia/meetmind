## User Story

**Como** Ricardo (líder de proyecto)  
**Quiero** ver la lista de participantes identificados en la reunión  
**Para** saber quién estuvo presente y asignar responsabilidades

---

## Descripción ampliada

El sistema debe extraer e identificar los nombres de las personas que participaron en la reunión a partir del texto transcrito o las notas. La extracción se realiza mediante un nodo especializado `extract_participants` que utiliza un prompt optimizado para:

- Identificar nombres propios en contexto de reunión
- Excluir términos genéricos ("persona", "alguien", "un participante")
- Manejar variantes y abreviaturas (ej. "Juan Pérez" vs "Juan")
- Devolver lista vacía o "No identificados" cuando no hay datos suficientes

La salida se almacena en el campo `participants` del State del grafo, como cadena separada por comas.

---

## Criterios de Aceptación

- [ ] Los participantes se muestran como lista de nombres separados por comas
- [ ] No se incluyen términos genéricos como "persona" o "alguien"
- [ ] Si no hay participantes identificables, se muestra lista vacía o "No identificados"
- [ ] Los nombres son coherentes con el contexto de la reunión

---

## Especificación BDD

```gherkin
Feature: Ver extracción de participantes
  Como Ricardo
  Quiero ver la lista de participantes identificados en la reunión
  Para saber quién estuvo presente y asignar responsabilidades

  Scenario: Reunión con participantes claros
    Given que el texto menciona "Juan, María y Pedro asistieron a la reunión"
    When el sistema procesa el texto
    Then la lista de participantes incluye Juan, María, Pedro
    And no incluye "persona" ni "alguien"

  Scenario: Texto sin participantes identificables
    Given que el texto no menciona nombres concretos
    When el sistema procesa el texto
    Then la salida es lista vacía o "No identificados"

  Scenario: Participantes con apellidos
    Given que el texto menciona "Laura García propuso..."
    When el sistema procesa el texto
    Then la lista incluye el nombre completo o la variante más usada
```
