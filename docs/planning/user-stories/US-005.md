## User Story

**Como** Ricardo o Pablo  
**Quiero** ver las acciones acordadas con su responsable asignado  
**Para** dar seguimiento a compromisos y trazabilidad

---

## Descripción ampliada

El sistema debe extraer las acciones acordadas en la reunión y, cuando sea posible, asignar un responsable a cada una. El nodo `extract_actions` debe:

- Identificar compromisos explícitos e implícitos
- Asociar responsable cuando sea identificable en el texto
- Usar "Por asignar" cuando no se pueda determinar el responsable
- Formatear la salida como "acción | responsable", separando cada par con pipe (|)

Esta salida es crítica para Ricardo (trazabilidad) y Pablo (referencias de acuerdos técnicos).

---

## Criterios de Aceptación

- [ ] Las acciones incluyen responsable cuando es identificable
- [ ] Formato "acción | responsable" o similar
- [ ] Si el responsable no es identificable, se muestra "Por asignar"
- [ ] Las acciones implícitas se incluyen con la mejor estimación posible
- [ ] Salida separada por pipe (|)

## Especificación BDD

```gherkin
Feature: Ver acciones acordadas con responsables
  Como Ricardo o Pablo
  Quiero ver las acciones acordadas con su responsable asignado
  Para dar seguimiento a compromisos y trazabilidad

  Scenario: Acciones con responsables explícitos
    Given que el texto dice "María enviará el informe antes del viernes"
    When el sistema procesa el texto
    Then la salida incluye "Enviar informe antes del viernes | María"
    And el formato es acción | responsable

  Scenario: Acción sin responsable
    Given que el texto dice "Se debe revisar el contrato"
    When el sistema procesa el texto
    Then la salida incluye "Revisar el contrato | Por asignar"

  Scenario: Múltiples acciones
    Given que hay varias acciones en el texto
    When el sistema extrae las acciones
    Then cada acción está separada por pipe
    And se incluyen acciones implícitas cuando son inferibles
```
