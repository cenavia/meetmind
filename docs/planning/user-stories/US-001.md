## User Story

**Como** Marina (asistente) o Pablo (desarrollador)  
**Quiero** subir una grabación de reunión (audio o video)  
**Para** obtener documentación estructurada sin transcribir manualmente

---

## Descripción ampliada

El sistema debe aceptar archivos multimedia (audio o video) de reuniones y, mediante un servicio de Speech-to-Text (STT), transcribirlos automáticamente. Una vez obtenido el texto, el workflow estándar (extracción de participantes, temas, acciones, minuta y resumen) se ejecuta sobre la transcripción.

El nodo `preprocess` debe detectar entradas multimedia y disparar la transcripción antes de pasar el texto al resto del pipeline. El usuario debe recibir feedback durante el procesamiento (indeterminado o mensaje de progreso), dado que la transcripción puede tardar varios segundos o minutos según la duración del archivo.

---

## Criterios de Aceptación

- [ ] El sistema acepta archivos MP4, MOV, MP3, WAV, M4A, WEBM, MKV
- [ ] La transcripción se ejecuta automáticamente y produce texto utilizable
- [ ] El workflow genera participantes, temas, acciones, minuta y resumen ejecutivo
- [ ] El usuario recibe feedback durante el procesamiento
