## User Story

**Como** Marina o Ricardo  
**Quiero** obtener una minuta formal estructurada de la reunión  
**Para** compartir documentación profesional con stakeholders

---

## Descripción ampliada

El sistema genera una minuta formal a partir de la información ya extraída (participantes, temas, acciones). El nodo `generate_minutes` debe:

- Producir un texto de máximo 150 palabras
- Usar tono profesional y estructura formal
- Integrar participantes, temas y acciones de forma coherente
- Mantener legibilidad y coherencia con el contenido original

La minuta es el artefacto principal que Marina comparte con el equipo y Ricardo usa para auditoría.

---

## Criterios de Aceptación

- [ ] La minuta tiene máximo 150 palabras
- [ ] Tono profesional y estructura formal
- [ ] Incluye la información de participantes, temas y acciones procesados
- [ ] Es legible y coherente con el contenido original

## Especificación BDD

```gherkin
Feature: Ver minuta formal
  Como Marina o Ricardo
  Quiero obtener una minuta formal estructurada de la reunión
  Para compartir documentación profesional con stakeholders

  Scenario: Generar minuta completa
    Given que existen participantes, temas y acciones extraídos
    When el sistema genera la minuta
    Then la minuta tiene máximo 150 palabras
    And usa tono profesional
    And incluye participantes, temas y acciones

  Scenario: Minuta con información parcial
    Given que solo se identificaron temas (sin participantes ni acciones)
    When el sistema genera la minuta
    Then la minuta integra la información disponible
    And mantiene coherencia y legibilidad
```
