## User Story

**Como** Marina o cualquier usuario  
**Quiero** recibir mensajes claros cuando la información es incompleta o hay errores  
**Para** entender qué pasó y qué puedo hacer al respecto

---

## Descripción ampliada

El sistema debe manejar de forma elegante los casos edge y errores:

- **Información incompleta:** Texto muy corto, sin participantes identificables, transcripción de baja calidad
- **Errores de procesamiento:** Fallos en transcripción, timeout del LLM, archivo corrupto
- **Estados parciales:** Resultados parciales (ej. solo temas, sin acciones)

El usuario nunca debe ver errores técnicos crudos (stack traces, mensajes de excepción). En su lugar: mensajes amigables, estado "partial" o "failed" con descripción legible, y campo `processing_errors` para advertencias.

---

## Criterios de Aceptación

- [ ] Mensajes claros cuando no se identifican participantes
- [ ] Advertencia cuando el texto es muy corto o de baja calidad
- [ ] Estado "partial" cuando hay resultados parciales
- [ ] Estado "failed" con descripción del error cuando falla el procesamiento
- [ ] Campos processing_errors para advertencias o errores
- [ ] No se muestran errores técnicos crudos al usuario final

```gherkin
    When el sistema procesa
    Then se incluye advertencia en processing_errors

  Scenario: Error de transcripción
    Given que la transcripción del audio falla
    When el usuario consulta el resultado
    Then el estado es "failed"
    And la descripción es legible ("No se pudo transcribir el audio")
    And no se expone el error técnico

  Scenario: Timeout del LLM
    Given que la llamada al modelo excede el timeout
    When ocurre el fallo
    Then el usuario recibe mensaje "Procesamiento interrumpido"
    And se sugiere reintentar
```

## Estrategia de robustez

1. **Validación en API:** Texto vacío, archivo no soportado, tamaño excesivo
2. **Validación en preprocess:** Longitud mínima de texto, calidad
3. **Try/except en nodos:** Capturar errores, registrar, devolver mensaje amigable
4. **Post-procesamiento:** Detectar resultados vacíos y etiquetar como partial si hay otros datos
