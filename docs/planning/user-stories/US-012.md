## User Story

**Como** sistema  
**Quiero** persistir cada reunión procesada en base de datos  
**Para** permitir consulta posterior y trazabilidad

---

## Descripción ampliada

Cada reunión procesada debe guardarse en base de datos con:

- **Id único (UUID):** Identificador inmutable
- **Datos de resultado:** participantes, temas, acciones, minuta, resumen
- **Metadatos:** archivo origen (nombre, tipo), fecha de creación (created_at)
- **Estado:** completed | failed | partial
- **Errores:** processing_errors (array o string) para advertencias o mensajes de fallo

Las operaciones CRUD necesarias: create (al finalizar procesamiento), get (por id), list (todas o paginadas). Los datos deben persistir entre reinicios.

---

## Criterios de Aceptación

- [ ] Cada reunión recibe id único (UUID)
- [ ] Se guardan: participantes, temas, acciones, minuta, resumen, archivo origen
- [ ] Se guardan: status (completed | failed | partial), created_at, processing_errors
- [ ] Operaciones create, get, list funcionan correctamente
- [ ] Los datos persisten entre reinicios de la aplicación

## Especificación BDD

```gherkin
Feature: Almacenar reuniones procesadas de forma persistente
  Como sistema
  Quiero persistir cada reunión procesada en base de datos
  Para permitir consulta posterior y trazabilidad

  Scenario: Guardar reunión procesada
    Given que el workflow completó exitosamente
    When se finaliza el procesamiento
    Then se crea un registro con UUID único
    And se guardan participantes, temas, acciones, minuta, resumen
    And se guarda status "completed" y created_at

  Scenario: Consultar reunión por id
    Given que existe una reunión con id "abc-123"
    When consulto get("abc-123")
    Then recibo el registro completo
    And los datos coinciden con lo guardado

  Scenario: Listar reuniones
    Given que existen 5 reuniones guardadas
    When consulto list()
    Then recibo las 5 reuniones
    And están ordenadas por created_at (desc)

  Scenario: Persistencia tras reinicio
    Given que guardé una reunión
    When reinicio la aplicación
    Then la reunión sigue disponible
    And get(id) retorna los mismos datos
```

## Modelo de datos

| Campo | Tipo | Descripción |
|-------|------|-------------|
| id | UUID | Clave primaria |

| topics | string | Lista separada por punto y coma |
| actions | string | Lista separada por pipe |
| minutes | string | Minuta formal |
| summary | string | Resumen ejecutivo |
| source_file | string? | Nombre del archivo origen |
| status | enum | completed, failed, partial |
| created_at | datetime | Fecha de creación |
| processing_errors | string? | Errores o advertencias |
