## User Story

**Como** Marina o Pablo  
**Quiero** subir un archivo de texto (TXT o Markdown) con notas de reunión  
**Para** obtener documentación estructurada sin transcripción

---

## Descripción ampliada

El sistema debe aceptar archivos de texto plano (TXT) o Markdown con notas tomadas durante o después de una reunión. A diferencia del flujo multimedia, aquí no hay transcripción: el contenido se lee directamente y se pasa al workflow de extracción y generación.

Esta es la ruta de entrada más simple y permite demos rápidas sin depender de servicios STT. El nodo `preprocess` debe detectar el tipo de entrada (texto) y leer el contenido del archivo respetando encoding (UTF-8 por defecto).

---

## Criterios de Aceptación

- [ ] El sistema acepta archivos TXT y Markdown
- [ ] El contenido pasa directamente al workflow sin transcripción
- [ ] Se generan las 5 salidas estructuradas (participantes, temas, acciones, minuta, resumen)
- [ ] Se respeta el formato y encoding del archivo

```gherkin
    And no se ejecuta transcripción

  Scenario: Subir archivo Markdown
    Given que tengo un archivo .md con notas en formato Markdown
    When subo el archivo al sistema
    Then el sistema procesa el contenido correctamente
    And genera las 5 salidas estructuradas

  Scenario: Archivo con encoding UTF-8
    Given que tengo un archivo con caracteres especiales (ñ, á, etc.)
    When subo el archivo
    Then el contenido se interpreta correctamente
```
