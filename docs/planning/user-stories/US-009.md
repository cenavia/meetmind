## User Story

**Como** Ricardo o Elena  
**Quiero** ver el historial de reuniones procesadas  
**Para** acceder a documentación pasada sin volver a procesar

---

## Descripción ampliada

Los usuarios deben poder consultar reuniones procesadas anteriormente. El historial incluye:

- Lista de reuniones con identificador único (id)
- Vista de detalle: participantes, temas, acciones, minuta, resumen
- Metadatos: archivo origen, fecha de procesamiento
- Estado: completed | failed | partial

El acceso puede realizarse desde la interfaz Gradio (lista + detalle) y/o vía API REST.

---

## Criterios de Aceptación

- [ ] Lista de reuniones procesadas con identificación única (id)
- [ ] Vista de detalle de cada reunión (participantes, temas, acciones, minuta, resumen)
- [ ] Información de archivo origen y fecha de procesamiento
- [ ] Estado de cada reunión (completed | failed | partial)
- [ ] Acceso desde la interfaz Gradio y/o API

## Especificación BDD

```gherkin
Feature: Consultar historial de reuniones procesadas
  Como Ricardo o Elena
  Quiero ver el historial de reuniones procesadas
  Para acceder a documentación pasada sin volver a procesar

    Given que existen reuniones procesadas
    When consulto el historial

  Scenario: Ver detalle de reunión
    Given que existe una reunión con id "abc-123"
    When selecciono esa reunión
    Then veo participantes, temas, acciones, minuta y resumen
    And veo archivo origen y fecha de procesamiento

  Scenario: Reunión con estado failed
    Given que una reunión falló en el procesamiento
    When consulto el historial
    Then el estado aparece como "failed"
    And hay descripción del error si está disponible
```
